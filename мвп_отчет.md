# Отчет по текущей версии проекта (MVP) системного анализа транспорта

## 1. Цель проекта и решаемая проблема
**Какую проблему решаем:** 
Отсутствие удобного и наглядного инструмента для анализа движения общественного транспорта, выявления проблемных участков маршрутов (заторов) и оценки скоростного режима в реальном времени или по историческим данным телеметрии. Диспетчерам и аналитикам транспортных компаний сложно "на глаз" определить паттерны задержек и системные пробки.

**Цель проекта:** 
Создать современное веб-приложение для загрузки пространственно-временных данных (выгрузок телеметрии), их обработки и всесторонней визуализации. Проект направлен на вычисление ключевых статистик поездок: средних и медианных скоростей, индекса заторов, а также визуальное выделение кластеров (эпицентров) транспортных задержек на тепловой и обычной интерактивных картах.

## 2. Сценарий работы программы (Workflow)
Сценарий работы построен от начальной загрузки данных к детализированному анализу:

1. **Загрузка данных:** Пользователь загружает датасет с точками треков транспортных средств (GeoJSON) через боковое меню. Приложение автоматически парсит файл и определяет доступный временной отрезок и список имеющихся маршрутов.
2. **Настройка параметров анализа:**
   - *Временной интервал:* указывается точное окно начала и конца.
   - *Фильтрация маршрутов:* выбор конкретных маршрутов из списка доступных.
   - *Тонкая настройка поиска заторов:* установка "порога затора" в км/ч, радиуса кластера (eps) и исключение/включение точек с нулевой скоростью (полезно, чтобы не считать остановки за пробки).
3. **Запуск анализа:** При нажатии "Анализировать" фронтенд формирует сводный пакет параметров и отправляет на backend (через REST API `/analyze`).
4. **Обработка (Backend):** Алгоритмы валидируют данные, фильтруют их в оперативной памяти и запускают алгоритм пространственной кластеризации (по всей видимости, на основе алгоритмов типа DBSCAN) для выявления "плотных" скоплений точек с низкой скоростью.
5. **Отображение результатов:** Построенные маршруты, многоугольники выявленных заторных зон, сводная статистика и график скоростей отрисовываются в интерфейсе.

## 3. Подробное описание данных

### Входные данные (Input)
Источником является `GeoJSON` формат (FeatureCollection). Каждая точка (Feature) содержит блок свойств `properties`:
- `time`: Отметка времени фиксации координаты транспортного средства.
- `speed`: Скорость движения (в км/ч).
- `route_num`: Номер маршрута (например, "80", "3").
- `rtype`: Тип транспортного средства (А - автобус, Т - трамвай, Тр - троллейбус, М - маршрутное такси).
- Дополнительная метаинформация: гос. номера (`gos_num` / `plate`), наличие инфраструктуры для инвалидов (`low_floor`), наличие сети (`wifi`).
- Настройки кластеризации и фильтрации: `speed_thresh` (порог скорости), `eps_m` (радиус кластеризации в метрах), `min_pts` (минимальное число точек).

### Выходные данные (Output)
После обработки бэкенд возвращает проанализированный объект:
- `filtered_geojson`: Очищенный от лишних или не попавших в фильтр (по маршруту/времени) точек список.
- `congestion_zones`: Массив полигонов (GeoJSON Feature), физически очерчивающих границы заторов на карте.
- `statistics`: Агрегированные метрики — средняя скорость, медиана и общее число валидных точек. 
- `congestion_index`: Оценка уровня затора (по условной 10-балльной шкале).
- `plot`: Массивы `times` и `speeds` для рендеринга профиля скорости во времени.

## 4. Описание интерфейса
Интерфейс приложения построен по современным принципам дашбордов и функционально разделен на три ключевых области:

### 1. Боковая панель управления (Sidebar)
Находится слева, может сворачиваться для расширения карты. Отвечает за входные параметры:
- **Загрузка файла:** Drop-зона или кнопка выбора `GeoJSON`.
- **Начало / Конец:** Ввод диапазона (интерфейс datetime) для фильтрации конкретных минут/часов пик.
- **Маршруты:** Чекбоксы с номерами загруженных маршрутов.
- **Включать 0 км/ч:** Позволяет исключить из расчета стоянки на остановках и светофорах.
- **Порог затора (слайдер):** Какую скорость считать индикатором "стояния" (по умолчанию порядка 8 км/ч).
- **Радиус кластера (слайдер):** Определяет, какое расстояние между медленными машинами считать одной общей пробкой (настройка радиуса в метрах).
- **Действия:** "Анализировать", "Очистить данные" и "Экспорт" (вызов системной функции печати страницы/вывод в PDF).

### 2. Главная интерактивная карта (Main Map)
Центральный блок дашборда, реализованный на библиотеке Leaflet.
- **Режим трекинга (по умолчанию):** Отображает все точки. Точки раскрашены градиентом в зависимости от скорости:
  - Красный: <10 км/ч (затор)
  - Оранжевый: 10 - 20 км/ч (затрудненное движение)
  - Желтый: 20 - 40 км/ч
  - Зеленый: >40 км/ч (свободное движение)
- **Информация по клику (Popup):** При нажатии на любую маркер-точку всплывает карточка с деталями: гос. номер рейса, скорость в момент фиксации, тип ТС, точное время, иконки доступности (Wi-Fi, Низкий пол).
- **Зоны заторов:** Выявленные бэкендом очаги пробок показываются поверх карты красными полупрозрачными многоугольниками.
- **Режим "Тепловой карты" (Heatmap):** Активируется круглой кнопкой справа вверху. Режим акцентирует внимание только на местах значительных задержек: чем больше в определенной области машин двигалось медленно, тем ярче ("краснее") цветовое термо-пятно. Позволяет мгновенно оценить самые проблемные перекрёстки дороги.

### 3. Панель графиков и статистики (Chart Section)
Располагается внизу экрана с возможностью изменения высоты путем перетаскивания (Resizer handle).
- **Ключевые показатели (KPI слева):**
  - *Ср. скорость* и *Медиана*: В километрах в час.
  - *Индекс затора*: От 1 до 10 (меняет цвет: зеленый-желтый-красный в зависимости от критичности значения).
  - *Зон заторов*: Динамический счетчик обнаруженных кластеров.
  - *Всего точек*: Объем выборки.
- **Профиль скорости (справа):**
  Линейный интерактивный график изменения скорости с течением времени. 
  - На оси X — время поездок (с шагом до минут).
  - На оси Y — скорость ТС (км/ч).
  - Поддерживается функционал зуммирования (приближение колесиком мыши) и панорамирования графика влево-вправо для детального изучения поведения маршрута на конкретных участках времени. Доступна кнопка "Сброс зума" для возвращения к исходному обзору.
